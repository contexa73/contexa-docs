Features

How It Works

A comprehensive security platform that unifies identity, authorization, and AI-driven threat analysis in a single Spring-native integration.

Identity DSL

Declarative authentication via IdentityDslRegistry. A single fluent chain — .form().session().build() — generates multiple SecurityFilterChain beans at runtime. Supports Form, REST, OTT, Passkey, MFA, and OAuth2.

Dynamic Authorization

Full XACML engine (PAP/PDP/PEP/PIP/PRP) with database-driven policy evaluation. @Protectable adds AI-driven method-level security — set sync = true for real-time AI verification on high-risk operations like payments.

AI Security Engine

HCAD (Hybrid Cold/Hot-path Anomaly Detection) with a 5-layer pipeline: Request → Strategy → Lab → Pipeline → Response. The Hot Path applies rule-based evaluation; the Cold Path escalates to 2-Layer LLM analysis for deep threat assessment.

Zero Trust

Continuous verification after authentication. The async AI pipeline analyzes every request in the background with zero latency impact, stores decisions in Redis, and ZeroTrustAccessControlFilter enforces them on the next request.

Shadow Mode

Safe 2-stage deployment controlled by security.zerotrust.mode. In SHADOW, the full AI pipeline runs but SecurityDecisionEnforcementHandler skips writing to Redis — observe and validate AI decisions before switching to ENFORCE for full production enforcement.

Streaming & RAG

Two execution modes: standard returns Mono<AIResponse> via a 6-step pipeline; streaming returns Flux<ServerSentEvent> for real-time output. PgVector-backed RAG provides context-aware threat analysis with cosine similarity search.

Infrastructure

Operating Modes

Choose the deployment topology that fits your environment. Switch between modes with a single configuration property.

Standalone

All Contexa components run within a single Spring Boot application. Uses in-memory caching and local event handling. Ideal for development, testing, and single-instance deployments.

  • In-memory cache with Caffeine
  • Local event publishing
  • Zero external dependencies
  • Default mode (no configuration needed)

Distributed

Contexa leverages Redis for shared caching, distributed sessions, and cross-instance event propagation. Designed for clustered and high-availability production environments.

  • Redis-backed shared cache
  • Cross-instance event propagation
  • Horizontal scaling support
  • Session sharing across nodes
YAML 
# Standalone (default — no external dependencies)
contexa:
  infrastructure:
    mode: standalone

# Distributed (requires Redis)
contexa:
  infrastructure:
    mode: distributed

spring:
  data:
    redis:
      host: localhost
      port: 6379
