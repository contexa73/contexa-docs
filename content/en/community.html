Contributing

Contexa is an open source project and we welcome contributions of all kinds — from bug fixes and documentation improvements to new features and architectural enhancements.

1

Fork & Clone

Fork the repository on GitHub and clone it to your local machine.

Shell 
git clone https://github.com/your-username/contexa.git
cd contexa
git checkout -b feature/your-feature-name
2

Build & Test

Ensure all existing tests pass before making changes. Contexa uses Gradle for builds.

Shell 
./gradlew build
./gradlew test
3

Submit a Pull Request

Push your branch and open a pull request against the main branch. Describe your changes clearly and reference any related issues.

Community vs Enterprise

Contexa Community Edition is free and open source under the Apache License 2.0. Enterprise Edition provides additional capabilities for production-critical workloads.

Feature Community Enterprise

XACML Policy Engine & @Protectable Included Included
Zero Trust Actions (5 actions) Included Included
HCAD AI Engine (Hot Path + Cold Path) Included Included
Shadow Mode (2-stage migration) Included Included
SSE Streaming & PgVector RAG Included Included
Standalone & Distributed modes Included Included
