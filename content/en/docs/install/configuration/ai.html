AI Engine Configuration

Configuration properties for the Contexa AI engine, including the tiered LLM strategy, RAG, advisor chain, streaming pipeline, and tuning scenarios.

Quick Start: Minimal Config

The minimum configuration to get the AI engine running with Ollama:

YAML 
spring:
  ai:
    ollama:
      base-url: http://localhost:11434
      chat:
        model: qwen2.5:7b
    security:
      layer1:
        model: qwen2.5:7b
      layer2:
        model: qwen2.5:7b
    vectorstore:
      pgvector:
        index-type: HNSW
        distance-type: COSINE_DISTANCE
        dimensions: 384
        initialize-schema: true

This sets up a single model for both tiers and configures pgvector for RAG. All other settings use sensible defaults.

Tuning Scenarios

Common configuration adjustments for specific situations:

Slow Responses

If LLM responses are too slow, reduce the model size or increase timeouts:

YAML 
spring:
  ai:
    security:
      layer1:
        model: qwen2.5:7b          # Smaller, faster model
      tiered:
        layer1:
          timeout-ms: 60000         # Increase timeout to 60s
          timeout:
            totalMs: 30000
            llmMs: 20000

RAG Results Are Inaccurate

If retrieved context is not relevant enough, tighten the similarity threshold and reduce top-K:

YAML 
contexa:
  rag:
    defaults:
      similarityThreshold: 0.85    # Stricter matching
      topK: 3                      # Fewer but more relevant docs
spring:
  ai:
    security:
      tiered:
        layer1:
          rag:
            similarityThreshold: 0.7  # Per-tier threshold

Token Usage / Cost Optimization

Reduce tokens by limiting prompt content:

YAML 
spring:
  ai:
    security:
      tiered:
        layer1:
          prompt:
            maxRagDocuments: 2          # Fewer RAG docs in prompt
            maxSimilarEvents: 1         # Fewer historical events
          vectorSearchLimit: 3          # Fewer vector search results
        truncation:
          layer1:
            payload: 100                # Shorter payload excerpts
            ragDocument: 150            # Shorter RAG doc excerpts

Production Deployment

Recommended settings for production with separate models per tier:

YAML 
spring:
  ai:
    security:
      layer1:
        model: qwen2.5:14b
        backup:
          model: llama3.2:latest
      layer2:
        model: exaone3.5:latest
        backup:
          model: deepseek-r1:14b
      tiered:
        vectorCache:
          maxSize: 50000
          expireMinutes: 10
          enabled: true
          recordStats: true
        security:
          trustedProxyValidationEnabled: true
contexa:
  advisor:
    security:
      enabled: true
      requireAuthentication: true
