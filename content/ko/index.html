Spring 을 위한 AI-Native 보안 플랫폼

Spring Security와 Spring AI를 핵심 엔진으로 구축된 보안 플랫폼.
모든 Spring 애플리케이션과 자연스럽게 통합됩니다.
인증 이후에도 지속적으로 보안 상태를 유지하는 AI-Native Zero Trust 환경을 제공합니다.

@SpringBootApplication
@EnableAISecurity
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

@Service
public class OrderService {

    @Protectable
    public Order getOrder(Long orderId) {
        return orderRepository.findById(orderId);
    }
}

어노테이션 2개면 충분합니다.

@EnableAISecurity
AI 기반 Zero Trust를 활성화합니다. 메인 클래스에 어노테이션 하나만 추가하면 전체 보호가 적용됩니다.
@Protectable
모든 메서드에 소유자 기반 접근 제어를 적용합니다. AI가 사용자의 기준 패턴(Baseline)과 비교하여 매 호출을 검증합니다.
기능

작동 방식

Identity, 인가, AI 기반 위협 분석을 하나의 Spring 네이티브 통합으로 제공하는 종합 보안 플랫폼입니다.

Identity DSL

IdentityDslRegistry를 통한 선언적 인증 설정. 하나의 플루언트 체인 — .form().session().build() — 으로 런타임에 여러 SecurityFilterChain 빈을 생성합니다. Form, REST, OTT, Passkey, MFA, OAuth2를 지원합니다.

동적 인가

데이터베이스 기반 정책 평가를 갖춘 완전한 XACML 엔진(PAP/PDP/PEP/PIP/PRP). @Protectable은 AI 기반 메서드 수준 보안을 추가하며 — 결제와 같은 고위험 작업에는 sync = true를 설정하여 실시간 AI 검증을 수행합니다.

AI 보안 엔진

5단계 파이프라인을 갖춘 HCAD(Hybrid Cold/Hot-path Anomaly Detection): Request -> Strategy -> Lab -> Pipeline -> Response. Hot Path는 규칙 기반 평가를 적용하고, Cold Path는 심층 위협 평가를 위해 2계층 LLM 분석으로 에스컬레이션합니다.

Zero Trust

인증 이후 지속적 검증. 비동기 AI 파이프라인이 지연 시간 영향 없이 백그라운드에서 모든 요청을 분석하고, 결정을 Redis에 저장하며, ZeroTrustAccessControlFilter가 다음 요청에서 이를 적용합니다.

ALLOW BLOCK CHALLENGE ESCALATE PENDING_ANALYSIS

Shadow Mode

security.zerotrust.mode로 제어되는 안전한 2단계 배포. SHADOW 모드에서는 전체 AI 파이프라인이 실행되지만 SecurityDecisionEnforcementHandler가 Redis에 쓰기를 건너뜁니다 — AI 결정을 관찰하고 검증한 후 ENFORCE로 전환하여 운영 환경에서 완전히 적용합니다.

Streaming & RAG

두 가지 실행 모드: 표준 모드는 6단계 파이프라인을 통해 Mono<AIResponse>를 반환하고, 스트리밍 모드는 실시간 출력을 위해 Flux<ServerSentEvent>를 반환합니다. PgVector 기반 RAG는 코사인 유사도 검색으로 컨텍스트 인식 위협 분석을 제공합니다.

운영 모드

환경에 맞는 배포 토폴로지를 선택하세요. 설정 속성 하나만 변경하면 모드를 전환할 수 있습니다.

Standalone

모든 Contexa 컴포넌트가 단일 Spring Boot 애플리케이션 내에서 실행됩니다. 인메모리 캐싱과 로컬 이벤트 처리를 사용합니다. 개발, 테스트, 단일 인스턴스 배포에 적합합니다.

  • Caffeine 인메모리 캐시
  • 로컬 이벤트 발행
  • 외부 의존성 없음
  • 기본 모드 (설정 불필요)

Distributed

Contexa가 Redis를 활용하여 공유 캐싱, 분산 세션, 인스턴스 간 이벤트 전파를 수행합니다. 클러스터 및 고가용성 운영 환경을 위해 설계되었습니다.

  • Redis 기반 공유 캐시
  • 인스턴스 간 이벤트 전파
  • 수평 확장 지원
  • 노드 간 세션 공유
YAML 
# Standalone (default — no external dependencies)
contexa:
  infrastructure:
    mode: standalone

# Distributed (requires Redis)
contexa:
  infrastructure:
    mode: distributed

spring:
  data:
    redis:
      host: localhost
      port: 6379
