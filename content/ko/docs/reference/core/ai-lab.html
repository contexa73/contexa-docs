AI Lab 시스템

AI Lab 시스템은 도메인별 AI 처리 유닛을 구축하기 위한 범용 추상화를 제공합니다. 각 Lab은 동기, 비동기, Streaming 실행 모드를 갖춘 집중된 AI 워크로드를 캡슐화합니다. Lab은 AILabFactory를 통해 Spring 애플리케이션 컨텍스트에서 탐색되고 관리됩니다.

개요

Lab 아키텍처는 템플릿 메서드 패턴을 따릅니다. AILab<Req, Res>를 구현하거나 (AbstractAILab을 확장하여) 처리 유닛을 정의한 후 Spring 빈으로 등록합니다. AILabFactory는 런타임에 타입별로 Lab을 찾으며, Strategy 레이어는 현재 DiagnosisType에 따라 적절한 Lab에 작업을 위임합니다.

각 Lab은 생성 시 고유 ID가 할당되며 세 가지 실행 모드를 지원합니다:

  • 동기 — process(Req)는 결과가 준비될 때까지 블로킹합니다.
  • 비동기 — processAsync(Req)는 비블로킹 실행을 위한 Mono<Res>를 반환합니다.
  • Streaming — processStream(Req)는 실시간 청크 전달을 위한 Flux<String>을 반환합니다.

AILab<Req, Res>

모든 Lab이 구현해야 하는 핵심 인터페이스입니다. 제네릭 파라미터는 Lab 도메인의 요청 및 응답 타입을 정의합니다.

public interface AILab<Req, Res>
getLabId() String
이 Lab 인스턴스에 할당된 고유 식별자를 반환합니다. AbstractAILab에서 UUID 기반 접두사를 사용하여 자동 생성됩니다.
getLabName() String
생성 시 설정된 이 Lab의 사람이 읽을 수 있는 이름을 반환합니다.
process(Req request) Res
요청을 동기적으로 처리하고 결과를 반환합니다. 실패 시 LabProcessingException을 던집니다.
request Req 도메인별 요청 객체.
processAsync(Req request) Mono<Res>
Project Reactor의 Mono를 사용하여 비동기적으로 요청을 처리합니다. AbstractAILab의 기본 구현은 동기 doProcess 호출을 래핑합니다.
request Req 도메인별 요청 객체.
processStream(Req request) Flux<String>
응답을 일련의 문자열 청크로 스트리밍합니다. supportsStreaming()이 true를 반환할 때만 사용 가능합니다.
request Req 도메인별 요청 객체.
supportsStreaming() boolean
이 Lab이 Streaming 실행을 지원하는지 여부를 반환합니다. 기본값은 false입니다. Streaming을 활성화하려면 오버라이드하세요.
isActive() boolean
이 Lab이 현재 활성 상태이고 처리에 사용 가능한지 여부를 반환합니다. 기본값은 true입니다.
canProcess(Req request) boolean
이 Lab이 주어진 요청을 처리할 수 있는지 확인합니다. 기본 구현은 요청이 null이 아니고 Lab이 활성 상태일 때 true를 반환합니다.
request Req 평가할 요청.

AbstractAILab<Req, Res>

모든 Lab 구현의 기본 클래스입니다. 검증, 전처리, 핵심 처리, 후처리를 위한 수명 주기 훅과 함께 템플릿 메서드 패턴을 제공합니다. 하위 클래스는 doProcess()만 구현하면 됩니다.

public abstract class AbstractAILab<Req, Res> implements AILab<Req, Res>

템플릿 메서드 수명 주기

validateRequest(request)
preProcess(request)
doProcess(request)
postProcess(request, result)
doProcess(Req request) Res
추상. 하위 클래스가 구현해야 하는 핵심 처리 로직입니다. 검증 및 전처리 후 템플릿 메서드 내에서 호출됩니다.
doProcessAsync(Req request) Mono<Res>
진정한 비동기 처리를 위해 오버라이드합니다. 기본값은 Mono.fromCallable()로 래핑된 doProcess()에 위임합니다.
doProcessStream(Req request) Flux<String>
Streaming 출력을 제공하려면 오버라이드합니다. 기본값은 UnsupportedOperationException을 던집니다. supportsStreaming()도 true를 반환하도록 오버라이드해야 합니다.
validateRequest(Req request) void
처리 전 요청을 검증합니다. 기본값은 null을 확인합니다. 도메인별 검증을 추가하려면 오버라이드하세요.
preProcess(Req request) void
doProcess() 전에 실행되는 훅. 기본적으로 아무 작업도 수행하지 않습니다.
postProcess(Req request, Res result) void
성공적인 처리 후 실행되는 훅. 기본적으로 아무 작업도 수행하지 않습니다.

내부 클래스: LabProcessingException

Lab 처리가 실패할 때 던지는 런타임 예외입니다. 원래 원인을 래핑하고 메시지에 Lab 이름을 포함합니다.

AILabFactory

Spring 애플리케이션 컨텍스트에서 Lab 인스턴스를 탐색하고 검색하기 위한 팩토리 인터페이스입니다.

public interface AILabFactory
getLab(Class<T> labType) Optional<T>
클래스 타입별로 Lab 빈을 검색합니다. Lab이 등록되지 않은 경우 Optional.empty()를 반환합니다.
createLab(Class<T> labType) T
기존 Lab을 반환하거나 존재하지 않으면 UnsupportedOperationException을 던집니다. Lab은 Spring 빈으로 사전 등록되어야 합니다.
getLabByClassName(String className) Optional<AILab<?, ?>>
정규화된 클래스 이름으로 Lab을 검색합니다. 설정 기반 동적 Lab 해석에 유용합니다.
hasLab(Class<?> labType) boolean
주어진 타입의 Lab이 등록되어 있는지 확인합니다. 기본 메서드는 getLab()에 위임합니다.

DefaultAILabFactory

Spring ApplicationContext를 사용하여 Lab 빈을 탐색하는 기본 AILabFactory 구현체입니다.

public class DefaultAILabFactory implements AILabFactory

이 클래스는 CoreAutonomousAutoConfiguration에 의해 자동 설정됩니다. ApplicationContext.getBean(Class)를 사용하여 Lab을 해석하므로, AILab을 구현하고 Spring 컴포넌트로 어노테이션된 모든 클래스가 자동으로 탐색 가능합니다.

코드 예제

커스텀 Lab 구현

Java 
@Component
public class ThreatAnalysisLab
        extends AbstractAILab<ThreatRequest, ThreatResponse> {

    private final UnifiedLLMOrchestrator orchestrator;

    public ThreatAnalysisLab(UnifiedLLMOrchestrator orchestrator) {
        super("ThreatAnalysis");
        this.orchestrator = orchestrator;
    }

    @Override
    protected ThreatResponse doProcess(ThreatRequest request)
            throws Exception {
        ExecutionContext ctx = ExecutionContext.builder()
                .prompt(request.toPrompt())
                .securityTaskType(SecurityTaskType.THREAT_FILTERING)
                .build();

        String json = orchestrator.execute(ctx).block();
        return ThreatResponse.fromJson(json);
    }

    @Override
    protected void validateRequest(ThreatRequest request) {
        super.validateRequest(request);
        if (request.getPayload() == null) {
            throw new IllegalArgumentException(
                    "Threat payload is required");
        }
    }
}

팩토리를 통한 Lab 검색

Java 
@Service
public class SecurityService {

    private final AILabFactory labFactory;

    public SecurityService(AILabFactory labFactory) {
        this.labFactory = labFactory;
    }

    public ThreatResponse analyzeThreat(ThreatRequest request) {
        ThreatAnalysisLab lab = labFactory
                .getLab(ThreatAnalysisLab.class)
                .orElseThrow(() -> new IllegalStateException(
                        "ThreatAnalysisLab not available"));

        return lab.process(request);
    }
}
