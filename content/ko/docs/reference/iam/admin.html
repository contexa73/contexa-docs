관리자 콘솔 (Admin Console)

보안 모니터링, 통합 정책 관리(Policy Center), 통합 ID 관리(Access Center), Zero Trust 운영, AI 기반 정책 생성을 제공하는 웹 기반 관리 콘솔입니다.

메뉴 구조

관리자 콘솔은 기능을 논리적 그룹으로 구성합니다. 일부 섹션은 에디션(Enterprise, SaaS)에 따라 조건부 표시됩니다.

메뉴 그룹페이지목적조건

Enterprise Operations Enterprise Home, Zero Trust Ops, SOAR 콘솔, 승인 수신함, MCP 도구, Permit 레지스트리, 실행 이력, 플레이북, 메트릭, 시스템 연동 엔터프라이즈급 보안 운영 Enterprise 전용
SaaS Control Plane 테넌트, 빌링, 전용 운영, 릴리스 거버넌스, 테넌트 워크스페이스 멀티테넌트 SaaS 플랫폼 관리 SaaS 전용

Zero Trust 실시간 현황, 차단된 사용자 실시간 보안 모니터링 및 차단 사용자 관리 항상
보안 IP 관리, 세션 관리, 비밀번호 정책 IP 접근 규칙, 활성 세션 제어, 비밀번호 정책 설정 항상

정책 센터 (Policy Center)

접근: /admin/policy-center

정책 센터는 모든 정책 관련 작업을 위한 통합 허브입니다. 2개의 메인 탭과 서브 섹션으로 구성됩니다.

생성 탭 (리소스 + 정책 생성)

생성 탭에는 리소스 워크벤치와 정책 생성 모드가 포함됩니다.

  • 리소스 워크벤치: 인라인 편집, 정책 설정 모달(간편 부여/AI 빌더), AJAX 액션(정의, 제외, 복원), 대규모 배포를 위한 서버사이드 페이지네이션.
  • 간편 모드: 검색 및 칩 기반 선택을 통한 역할-권한 매핑. 기본 ALLOW 정책을 생성합니다.
  • 수동 모드: 동적 Target(URL/METHOD), Rule, Condition(SpEL) 편집을 포함한 전체 정책 폼. Policy/PolicyTarget/PolicyRule/PolicyCondition 엔티티를 직접 생성합니다.
  • AI 모드: SSE 스트리밍을 활용한 자연어 정책 생성. AI가 기존의 모든 역할, 권한, 조건, 정책을 분석합니다. 시스템 통계와 조건 템플릿 검색을 포함하여 컨텍스트를 제공합니다.

목록 탭

키워드 검색을 포함한 서버사이드 페이지네이션 정책 목록. 수정 및 삭제 액션.

실시간 보안 현황

접근: /admin/security-monitor

audit_log 테이블 기반의 실시간 보안 이벤트 뷰어. 모든 쿼리는 수백만 건을 처리할 수 있도록 DB 레벨 페이지네이션을 사용합니다.

대시보드 드릴다운

대시보드의 보안 지표를 클릭하면 사전 적용된 필터와 함께 보안 모니터로 이동합니다:

대시보드 지표필터쿼리
근무 외 접근AFTER_HOURS09:00-18:00 외 또는 주말 이벤트
AI 보안 분석SECURITY_DECISIONeventCategory = 'SECURITY_DECISION'
관리자 오버라이드ADMIN_OVERRIDEeventCategory = 'ADMIN_OVERRIDE'
고유 IPDISTINCT_IPIP 그룹핑 (GROUP BY + count)
평균 위험도HIGH_RISKriskScore >= 0.4
AI 보안 분석 오류SECURITY_ERROReventCategory = 'SECURITY_ERROR'
접근 허용DECISION_ALLOWdecision = 'ALLOW'
접근 거부DECISION_DENYdecision = 'DENY' 또는 'BLOCK'

대시보드

대시보드(/admin/dashboard)는 최근 24시간 동안의 audit_log에서 집계된 데이터로 종합 보안 개요를 제공합니다.

섹션

  • 정책 현황: 전체 정책 수, 활성 리소스, 리소스 보호 범위
  • Zero Trust 판정: ALLOW/CHALLENGE/BLOCK/ESCALATE/DENY 분류의 도넛 차트
  • 인증 통계: 성공/실패 비율을 보여주는 반원 게이지
  • 보안 지표: 드릴다운 링크가 포함된 2열 그리드 -- 근무 외 접근, AI 분석, 관리자 오버라이드, 고유 IP, 평균 위험도, AI 보안 분석 오류
  • 위협 이벤트: 보안 모니터 상세 링크가 포함된 최근 고위험 이벤트
  • 차단된 사용자: Zero Trust 엔진에 의해 최근 차단된 사용자

접근 센터 (Access Center)

접근: /admin/access-center

단일 탭 뷰에서 전체 ID 체인(사용자 -> 그룹 -> 역할 -> 권한)을 관리하는 통합 인터페이스입니다.

사용자 탭

사용자를 검색하고 그룹/역할 배정을 관리합니다. 사용자 상세에서 그룹을 통해 상속된 권한과 직접 역할 배정을 확인할 수 있습니다.

엔드포인트메서드설명
/admin/access-center/api/usersGET키워드로 사용자 검색
/admin/access-center/api/users/{userId}/detailGET그룹, 역할, 권한을 포함한 사용자 상세
/admin/access-center/api/users/{userId}/groupsPOST사용자 그룹 배정 업데이트
/admin/access-center/api/users/{userId}/rolesPOST사용자 직접 역할 배정 업데이트

그룹 탭

배정된 역할과 멤버를 포함한 모든 그룹을 조회합니다. 그룹-역할 매핑을 업데이트합니다.

엔드포인트메서드설명
/admin/access-center/api/groupsGET전체 그룹 목록
/admin/access-center/api/groups/{groupId}/detailGET역할과 멤버를 포함한 그룹 상세
/admin/access-center/api/groups/{groupId}/rolesPOST그룹 역할 배정 업데이트

역할 탭

권한과 배정된 사용자를 포함한 모든 역할을 조회합니다. 역할-권한 매핑을 업데이트합니다.

엔드포인트메서드설명
/admin/access-center/api/rolesGET전체 역할 목록
/admin/access-center/api/roles/{roleId}/detailGET권한과 사용자를 포함한 역할 상세
/admin/access-center/api/roles/{roleId}/permissionsPOST역할 권한 배정 업데이트

ID 및 접근 관리

사용자(/admin/users), 그룹(/admin/groups), 역할(/admin/roles), 권한(/admin/permissions)에 대한 표준 CRUD 작업입니다.

접근 체인: 사용자 -> 그룹 -> 역할 -> 권한. 사용자는 그룹 멤버십과 역할 배정을 통해 권한을 상속받습니다.

역할 계층

접근: /admin/role-hierarchies

상위 역할은 하위 역할의 모든 권한을 자동으로 상속합니다. 계층은 Spring Security의 RoleHierarchy에 의해 해석되며, CustomDynamicAuthorizationManager 런타임 평가에서 고려됩니다.

Zero Trust 운영

실시간 보안 현황

접근: /admin/security-monitor

시간 범위 필터(1시간/6시간/24시간/3일), 카테고리 필터(AI 분석, 인증 실패, 관리자 오버라이드), 수백만 건을 위한 DB 레벨 페이지네이션을 포함한 실시간 감사 로그 뷰어.

차단된 사용자

접근: /admin/blacklist

Zero Trust 엔진에 의해 차단된 사용자를 관리합니다. 상태별 필터링(BLOCKED, UNBLOCK_REQUESTED, RESOLVED, TIMEOUT_RESPONDED)과 관리자 해제 워크플로우를 지원합니다.

보안 관리

IP 관리

접근: /admin/ip-management

ALLOW/DENY 유형의 IP 기반 접근 규칙을 관리합니다. IPv4/IPv6 주소 및 CIDR 표기법을 지원합니다. 규칙에 선택적 만료일을 설정하고 켜기/끄기를 전환할 수 있습니다. 규칙을 CSV로 내보낼 수 있습니다.

세션 관리

접근: /admin/session-management

페이지네이션과 검색을 포함한 활성 사용자 세션 모니터링. 세션 ID, 사용자명, 클라이언트 IP, User Agent, 로그인 시간, 마지막 활동을 표시합니다. 강제 세션 무효화(단일 세션 또는 사용자의 모든 세션)를 지원합니다. 세션을 CSV로 내보낼 수 있습니다.

비밀번호 정책

접근: /admin/password-policy

비밀번호 정책 규칙 설정: 최소/최대 길이, 대문자 필수, 소문자 필수, 숫자 필수, 특수문자 필수. 사용자 등록 및 비밀번호 변경 시 정책 규칙이 적용됩니다.

AI 정책 생성

AI 정책 생성 시스템은 SSE 스트리밍을 활용하여 자연어 요구사항으로부터 정책을 생성합니다.

API 엔드포인트

엔드포인트메서드설명
/admin/api/ai/policies/generate/streamPOST자연어 기반 SSE 스트리밍 정책 생성
/admin/api/ai/policies/generatePOST동기식 정책 생성
/admin/api/policies/build-from-business-rulePOST생성된 정책을 데이터베이스에 저장

동작 원리

  1. 사용자가 자연어 요구사항을 입력합니다 (예: "개발팀에게 /api/** 접근을 업무시간에만 허용")
  2. AdvancedPolicyGenerationLab이 데이터베이스에서 사용 가능한 모든 역할, 권한, 조건으로 요청을 확장합니다
  3. PolicyGenerationContextRetriever가 벡터 DB에서 유사한 정책 패턴을 검색합니다 (RAG)
  4. AI가 roleIds, permissionIds, conditions, ID-이름 매핑을 포함한 PolicyResponse를 생성합니다
  5. 프론트엔드에서 사용 가능한 항목과 대조하여 응답을 검증하고 미리보기를 렌더링합니다
  6. 사용자가 확인하면 BusinessPolicyService가 Policy + PolicyTarget + PolicyRule + PolicyCondition 엔티티를 생성합니다
  7. CustomDynamicAuthorizationManager가 리로드되어 정책이 즉시 적용됩니다

시작하기

배포부터 완전한 동적 인가까지의 전체 과정:

단계작업상세
1애플리케이션 배포리소스 스캐너가 모든 엔드포인트를 자동으로 탐지합니다
2정책 센터 열기 (리소스 탭)인라인 편집으로 탐지된 리소스를 검토합니다
3권한 정의인가가 필요한 리소스에 "권한 정의 & 정책 설정"을 클릭합니다
4정책 생성간편 모드(역할-권한 매핑), 수동 모드(Target/Rule/Condition), 또는 AI 모드(자연어)를 사용합니다
5검증정책이 핫 리로드를 통해 즉시 적용됩니다. 다양한 사용자 역할로 테스트합니다.
6모니터링대시보드와 실시간 보안 현황으로 보안 가시성을 확보합니다
