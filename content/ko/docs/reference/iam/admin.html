관리자 콘솔 (Admin Console)

보안 모니터링, 통합 정책 관리(Policy Center), ID 관리(사용자/그룹/역할/권한), Zero Trust 운영, 시각적 Authorization Studio를 제공하는 웹 기반 관리 콘솔입니다.

메뉴 구조

관리자 콘솔은 기능을 논리적 그룹으로 구성합니다. 일부 섹션은 에디션(Enterprise, SaaS)에 따라 조건부 표시됩니다.

정책 센터 (Policy Center)

접근: /admin/policy-center

정책 센터는 모든 정책 관련 작업을 위한 통합 허브입니다. 리소스, 정책 생성, 정책 목록, 분석 도구를 하나의 4탭 인터페이스로 결합합니다.

탭 1: 리소스

인라인 편집, 정책 설정 모달(간편 부여/AI 빌더), AJAX 액션(정의, 제외, 복원)을 포함한 전체 리소스 워크벤치. 대규모 배포를 위한 서버사이드 페이지네이션.

탭 2: 정책 생성 (3가지 모드)

  • 간편 모드: 검색 및 칩 기반 선택을 통한 역할-권한 매핑. 보다 복잡한 정책의 전제 조건으로 기본 ALLOW 정책을 생성합니다. 서버사이드 검색 API가 수천 개의 역할/권한을 처리합니다.
  • 수동 모드: 동적 Target(URL/METHOD), Rule, Condition(SpEL) 편집을 포함한 전체 정책 폼. Policy/PolicyTarget/PolicyRule/PolicyCondition 엔티티를 직접 생성합니다.
  • AI 모드: SSE 스트리밍을 활용한 자연어 정책 생성. AI가 기존의 모든 역할, 권한, 조건, 정책을 분석하여 최적의 정책 구성을 제안합니다. 결과는 검증 후 미리보기를 거쳐 저장됩니다.

탭 3: 정책 목록

키워드 검색을 포함한 서버사이드 페이지네이션 정책 목록. 수정 및 삭제 액션.

탭 4: 스튜디오

Authorization Studio, 정책 마법사, 리소스 워크벤치(Enterprise), 요약 통계로의 링크.

실시간 보안 현황

접근: /admin/security-monitor

audit_log 테이블 기반의 실시간 보안 이벤트 뷰어. 모든 쿼리는 수백만 건을 처리할 수 있도록 DB 레벨 페이지네이션을 사용합니다.

대시보드 드릴다운

대시보드의 보안 지표를 클릭하면 사전 적용된 필터와 함께 보안 모니터로 이동합니다:

대시보드

대시보드(/admin/dashboard)는 최근 24시간 동안의 audit_log에서 집계된 데이터로 종합 보안 개요를 제공합니다.

섹션

  • 정책 현황: 전체 정책 수, 활성 리소스, 리소스 보호 범위
  • Zero Trust 판정: ALLOW/CHALLENGE/BLOCK/ESCALATE/DENY 분류의 도넛 차트
  • 인증 통계: 성공/실패 비율을 보여주는 반원 게이지
  • 보안 지표: 드릴다운 링크가 포함된 2열 그리드 -- 근무 외 접근, AI 분석, 관리자 오버라이드, 고유 IP, 평균 위험도, AI 보안 분석 오류
  • 위협 이벤트: 보안 모니터 상세 링크가 포함된 최근 고위험 이벤트
  • 차단된 사용자: Zero Trust 엔진에 의해 최근 차단된 사용자

Authorization Studio

접근: /admin/studio

인가 분석을 위한 3패널 시각화 도구:

  • Explorer 패널: 사용자, 그룹, 역할 검색 및 탐색
  • Canvas 패널: AI 자연어 질의 (SSE 스트리밍) + Cytoscape 그래프 시각화 (네트워크/계층구조/플로우차트 레이아웃)
  • Inspector 패널: 상세 분석 결과, 유효 권한, 추천 사항

ID 및 접근 관리

사용자(/admin/users), 그룹(/admin/groups), 역할(/admin/roles), 권한(/admin/permissions)에 대한 표준 CRUD 작업입니다.

접근 체인: 사용자 -> 그룹 -> 역할 -> 권한. 사용자는 그룹 멤버십과 역할 배정을 통해 권한을 상속받습니다.

역할 계층

접근: /admin/role-hierarchies

상위 역할은 하위 역할의 모든 권한을 자동으로 상속합니다. 계층은 Spring Security의 RoleHierarchy에 의해 해석되며, Authorization Studio 분석과 CustomDynamicAuthorizationManager 런타임 평가 모두에서 고려됩니다.

Zero Trust 운영

실시간 보안 현황

접근: /admin/security-monitor

시간 범위 필터(1시간/6시간/24시간/3일), 카테고리 필터(AI 분석, 인증 실패, 관리자 오버라이드), 수백만 건을 위한 DB 레벨 페이지네이션을 포함한 실시간 감사 로그 뷰어.

차단된 사용자

접근: /admin/blacklist

Zero Trust 엔진에 의해 차단된 사용자를 관리합니다. 상태별 필터링(BLOCKED, UNBLOCK_REQUESTED, RESOLVED, TIMEOUT_RESPONDED)과 관리자 해제 워크플로우를 지원합니다.

워크플로우 마법사

정책 마법사

접근: /admin/policy-wizard

3단계 가이드 워크플로우: 역할 선택 -> 권한 선택 -> 검토 및 확인. 자동 생성 ALLOW 정책과 함께 역할-권한 매핑을 생성합니다.

권한 부여 마법사

접근: /admin/granting-wizard

실시간 영향 분석을 포함한 멤버십 관리. 변경사항 적용 전에 권한 획득/상실을 미리 보여줍니다.

AI 정책 생성

AI 정책 생성 시스템은 SSE 스트리밍을 활용하여 자연어 요구사항으로부터 정책을 생성합니다.

API 엔드포인트

엔드포인트메서드설명
/api/ai/policies/generate/streamPOST자연어 기반 SSE 스트리밍 정책 생성
/api/ai/policies/generatePOST동기식 정책 생성
/api/policies/build-from-business-rulePOST생성된 정책을 데이터베이스에 저장

동작 원리

  1. 사용자가 자연어 요구사항을 입력합니다 (예: "개발팀에게 /api/** 접근을 업무시간에만 허용")
  2. AdvancedPolicyGenerationLab이 데이터베이스에서 사용 가능한 모든 역할, 권한, 조건으로 요청을 확장합니다
  3. PolicyGenerationContextRetriever가 벡터 DB에서 유사한 정책 패턴을 검색합니다 (RAG)
  4. AI가 roleIds, permissionIds, conditions, ID-이름 매핑을 포함한 PolicyResponse를 생성합니다
  5. 프론트엔드에서 사용 가능한 항목과 대조하여 응답을 검증하고 미리보기를 렌더링합니다
  6. 사용자가 확인하면 BusinessPolicyService가 Policy + PolicyTarget + PolicyRule + PolicyCondition 엔티티를 생성합니다
  7. CustomDynamicAuthorizationManager가 리로드되어 정책이 즉시 적용됩니다

시작하기

배포부터 완전한 동적 인가까지의 전체 과정:

단계작업상세
1애플리케이션 배포리소스 스캐너가 모든 엔드포인트를 자동으로 탐지합니다
2정책 센터 열기 (리소스 탭)인라인 편집으로 탐지된 리소스를 검토합니다
3권한 정의인가가 필요한 리소스에 "권한 정의 & 정책 설정"을 클릭합니다
4정책 생성간편 모드(역할-권한 매핑), 수동 모드(Target/Rule/Condition), 또는 AI 모드(자연어)를 사용합니다
5검증정책이 핫 리로드를 통해 즉시 적용됩니다. 다양한 사용자 역할로 테스트합니다.
6모니터링대시보드와 실시간 보안 현황으로 보안 가시성을 확보합니다
