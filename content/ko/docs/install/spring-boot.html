Contexa가 Spring Boot에서 작동하는 방식

@EnableAISecurity는 Spring Security 위에 AI 기반 보안 파이프라인 전체를 활성화합니다. 이 페이지는 내부에서 무슨 일이 일어나는지, 그리고 기본 설정이 부족할 때 어떻게 커스터마이징하는지 설명합니다.

@EnableAISecurity가 활성화하는 것

애플리케이션에 @EnableAISecurity를 추가하면 시작 시 다음 체인이 실행됩니다:

  1. @EnableAISecurity가 AiSecurityImportSelector를 임포트하고, auto-configure 모듈에서 AiSecurityConfiguration을 로드합니다.
  2. AiSecurityConfiguration이 기존 PlatformConfig 빈을 확인합니다. 없으면(@ConditionalOnMissingBean) IdentityDslRegistry를 사용하여 기본 설정을 자동 생성합니다.
  3. PlatformConfig의 존재가 IdentitySecurityCoreAutoConfiguration을 트리거하여 전체 Zero Trust 필터 체인 — HCAD 필터, 접근 제어 필터, 챌린지 필터, 이벤트 퍼블리셔 — 을 등록합니다.

기본 PlatformConfig가 제공하는 것:

  • Form Login + OTT를 MFA 인증 흐름으로
  • 세션 관리 기본값
  • AISessionSecurityContextRepository — 보안 컨텍스트에 AI 신뢰 점수를 주입

커스터마이징이 필요하지 않을 수 있습니다. 기본 설정은 프로덕션 수준입니다. 아래 섹션은 인증 흐름, 인가 규칙 또는 세션 동작을 변경해야 할 때만 필요합니다.

요청 라이프사이클(Request Lifecycle)

모든 HTTP 요청은 다음 레이어를 순서대로 통과합니다. 전통적인 Spring Security는 인증에서 끝나지만, Contexa는 매 요청마다 AI 기반 행동 분석을 계속합니다.

필터 / 컴포넌트 역할
HCADFilter 행동 신호 추출 — 세션 상태, 디바이스 핑거프린트, IP, API 접근 패턴, 요청 속도
ZeroTrustAccessControlFilter 저장소에서 이전 AI 결정을 읽고 적용 (ALLOW, BLOCK, CHALLENGE, ESCALATE)
ZeroTrustChallengeFilter 결정이 CHALLENGE면 진행 전 MFA 재인증으로 리다이렉트
CustomDynamicAuthorizationManager XACML 정책 엔진을 통한 URL 레벨 접근 평가. 동적 인가 →
@Protectable Interceptor 메서드 실행 전 메서드 레벨 AI 인가 평가. @Protectable 참조 →
메서드 실행 비즈니스 로직이 정상적으로 실행됩니다
ZeroTrustEventPublisher 비동기 이벤트 발행 → AI가 백그라운드에서 요청 분석 → 결정이 저장되어 다음 요청에 적용

지연 시간 영향 없음. AI 분석은 요청 완료 후 비동기로 실행됩니다. 현재 요청은 AI에 의해 차단되지 않으며, 결정은 사용자의 다음 요청에 적용됩니다. 전체 Zero Trust 흐름 →

인증 커스터마이징

기본 인증 흐름을 변경하려면 PlatformConfig 빈을 직접 정의하세요. 각 IdentityDslRegistry 메서드는 독립적인 SecurityFilterChain을 생성합니다:

메서드 생성 대상 용도
global()공통 HTTP 설정CSRF, 인가 규칙, 세션 컨텍스트
form()Form Login 체인웹 UI 인증
rest()Stateless API 체인REST API 인증
mfa()Multi-factor 체인Form + OTT / Passkey 조합
ott()One-Time Token 체인매직 링크 인증
passkey()WebAuthn 체인패스키(Passkey) 인증
session()세션 관리세션 상태 및 라이프사이클

REST API 전용

Java 
return registry
    .global(globalHttpCustomizer)
    .rest(rest -> rest.order(10))
    .session(Customizer.withDefaults())
    .build();

Form + MFA + Passkey

Java 
return registry
    .global(globalHttpCustomizer)
    .mfa(mfa -> mfa
        .primaryAuthentication(auth -> auth
            .formLogin(form -> form.defaultSuccessUrl("/dashboard")))
        .passkey(Customizer.withDefaults())
    )
    .session(Customizer.withDefaults())
    .build();

인가 커스터마이징

global() 커스터마이저를 사용하여 URL 레벨 인가 규칙을 정의하세요. 불필요한 LLM 호출을 방지하기 위해 정적 리소스와 공개 API는 AI 평가에서 제외해야 합니다:

Java 
SafeHttpCustomizer<HttpSecurity> globalHttpCustomizer = http -> {
    http
        .csrf(AbstractHttpConfigurer::disable)
        .authorizeHttpRequests(authReq -> authReq
            .requestMatchers("/css/**", "/js/**", "/images/**", "/favicon.ico").permitAll()
            .requestMatchers("/api/public/**").permitAll()
            .anyRequest().access(customDynamicAuthorizationManager)
        )
        .securityContext(sc -> sc.securityContextRepository(aiSessionSecurityContextRepository));
};

메서드 레벨 보호가 필요하면 개별 메서드에 @Protectable을 추가하세요. AI가 메서드 실행 전 인가를 평가합니다.

AI 세션 컨텍스트

AISessionSecurityContextRepository는 Spring Security의 기본 컨텍스트 저장소를 대체합니다. 모든 세션에 AI가 계산한 신뢰 점수, 행동 기준선, 위협 평가를 추가하여 요청 간 지속적인 Zero Trust 평가를 가능하게 합니다.

Zero Trust 필수 요소. 커스텀 PlatformConfig를 정의할 경우, global() 커스터마이저에서 .securityContext(sc -> sc.securityContextRepository(aiSessionSecurityContextRepository))를 반드시 등록해야 합니다. 이것이 없으면 Zero Trust가 작동하지 않습니다. 상태 관리 →

전체 예제

REST API, Form Login with MFA and Passkey, 정적 리소스 제외, AI 세션 컨텍스트를 결합한 프로덕션 수준 설정:

Java 
@Configuration
@RequiredArgsConstructor
@EnableWebSecurity
public class PlatformSecurityConfig {

    private final CustomDynamicAuthorizationManager customDynamicAuthorizationManager;
    private final AISessionSecurityContextRepository aiSessionSecurityContextRepository;

    @Bean
    public PlatformConfig platformDslConfig(IdentityDslRegistry<HttpSecurity> registry) throws Exception {
        return registry
            .global(http -> http
                .csrf(AbstractHttpConfigurer::disable)
                .authorizeHttpRequests(authReq -> authReq
                    .requestMatchers("/css/**", "/js/**", "/images/**", "/favicon.ico").permitAll()
                    .requestMatchers("/api/public/**").permitAll()
                    .anyRequest().access(customDynamicAuthorizationManager))
                .securityContext(sc -> sc.securityContextRepository(aiSessionSecurityContextRepository)))
            .rest(rest -> rest.order(10))
            .mfa(mfa -> mfa
                .primaryAuthentication(auth -> auth
                    .formLogin(form -> form.defaultSuccessUrl("/dashboard")))
                .passkey(Customizer.withDefaults()))
            .session(Customizer.withDefaults())
            .build();
    }
}
