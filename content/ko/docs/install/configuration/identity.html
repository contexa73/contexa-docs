Identity 구성

인증 컨텍스트, 토큰 관리, MFA (Multi-Factor Authentication), 상태 머신 엔진을 포함한 Contexa Identity 모듈의 구성 속성입니다.

Auth Context 속성

spring.auth 접두사 아래의 속성으로, AuthContextProperties에 바인딩됩니다. 상태 타입, 토큰 전송, 토큰 생명주기, 동시 로그인 정책을 포함한 인증 컨텍스트를 제어합니다.

핵심 인증 설정

속성 타입 기본값 설명
spring.auth
.stateType StateType OAUTH2 인증 상태 관리 전략. SESSION은 서버측 세션 사용; OAUTH2는 토큰 기반 OAuth 2.0 흐름 사용
.tokenTransportType TokenTransportType HEADER 클라이언트와 서버 간 토큰 전송 방식. HEADER는 Authorization 헤더 사용; COOKIE는 HTTP 쿠키 사용
.accessTokenValidity long 3600000 액세스 토큰 유효 기간 (밀리초, 기본값: 1시간)
.refreshTokenValidity long 604800000 리프레시 토큰 유효 기간 (밀리초, 기본값: 7일)
.enableRefreshToken boolean true 액세스 토큰과 함께 리프레시 토큰 발급 여부
.allowMultipleLogins boolean false 사용자가 여러 동시 활성 세션을 가질 수 있는지 여부
.maxConcurrentLogins int 3 사용자당 최대 동시 로그인 세션 수 (allowMultipleLogins가 true일 때 적용)
.cookieSecure boolean true 인증 쿠키에 HTTPS 전송 필요 여부
.tokenPersistence String "memory" 토큰 영속성 전략. 지원값: memory, redis, jdbc
.oauth2Csrf boolean false OAuth2 엔드포인트에 대한 CSRF 보호 활성화 여부
YAML 
spring:
  auth:
    stateType: OAUTH2
    tokenTransportType: HEADER
    accessTokenValidity: 3600000
    refreshTokenValidity: 604800000
    enableRefreshToken: true
    allowMultipleLogins: false
    maxConcurrentLogins: 3
    cookieSecure: true
    tokenPersistence: memory
    oauth2Csrf: false

참고: AuthContext 참조

URL 구성

다양한 인증 흐름의 엔드포인트 경로를 정의하는 spring.auth.urls 아래의 중첩 URL 구성 객체입니다.

속성 타입 기본값 설명
spring.auth.urls
.single SingleAuthUrls - 단일 인증 흐름의 로그인 및 로그아웃 URL 경로
.primary PrimaryAuthUrls - 기본 인증 엔드포인트의 URL 경로
.mfa MfaUrls - MFA 챌린지, 검증, 등록 엔드포인트의 URL 경로
.factors FactorUrls - 개별 팩터별 엔드포인트(SMS, 이메일, TOTP)의 URL 경로

참고: 인증 URL 참조

OAuth2 토큰 설정

spring.auth.oauth2 아래의 중첩 설정으로, OAuth2TokenSettings에 바인딩됩니다. OAuth 2.0 인가 서버 연결, 클라이언트 자격 증명, JWK 키 스토어를 구성합니다.

속성 타입 기본값 설명
spring.auth.oauth2
.clientId String "default-client" 인가 서버에 등록된 OAuth2 클라이언트 식별자
.clientSecret String (자동 생성) OAuth2 클라이언트 시크릿. 지정하지 않으면 시작 시 자동 생성
.issuerUri String "http://localhost:9000" OAuth2 인가 서버 발급자의 기본 URI
.tokenEndpoint String "/oauth2/token" 인가 서버의 토큰 엔드포인트 경로
.scope String "read" 토큰 교환 시 요청되는 기본 OAuth2 스코프
.redirectUri String "http://localhost:8080" OAuth2 인가 성공 후 리디렉션할 URI
.authorizedUri String null 인가 후 리디렉션 URI (설정 시 redirectUri를 재정의)
.jwkKeyStorePath String null JWK 키 스토어의 파일 시스템 경로 (PKCS12 또는 JKS 형식)
.jwkKeyStorePassword String null JWK 키 스토어 파일의 비밀번호
.jwkKeyAlias String null JWK 키 스토어 내 키 항목의 별칭
.jwkKeyPassword String null 특정 키 항목의 비밀번호 (키 스토어 비밀번호와 다른 경우)
YAML 
spring:
  auth:
    oauth2:
      clientId: my-client-app
      clientSecret: ${OAUTH2_CLIENT_SECRET}
      issuerUri: https://auth.example.com
      tokenEndpoint: /oauth2/token
      scope: read
      redirectUri: https://app.example.com/callback
      authorizedUri: https://app.example.com/dashboard
      jwkKeyStorePath: /etc/contexa/keys/jwk-keystore.p12
      jwkKeyStorePassword: ${JWK_STORE_PASSWORD}
      jwkKeyAlias: contexa-signing-key
      jwkKeyPassword: ${JWK_KEY_PASSWORD}

참고: OAuth2 토큰 참조

MFA 타임아웃 및 생명주기

spring.auth.mfa 아래의 중첩 설정으로, MfaSettings에 바인딩됩니다. MFA 흐름의 세션 타임아웃, 챌린지 창, 장치 기억 기간을 제어합니다.

속성 타입 기본값 설명
spring.auth.mfa
.sessionTimeoutMs long 600000 MFA 세션 타임아웃 (밀리초, 기본값: 10분). 이 기간 내에 완료되지 않으면 세션 만료
.challengeTimeoutMs long 300000 개별 MFA 챌린지 타임아웃 (밀리초, 기본값: 5분). 각 챌린지는 이 기간 내에 응답해야 함
.inactivityTimeout long 900000 비활성 타임아웃 (밀리초, 기본값: 15분). 이 기간 동안 활동이 없으면 MFA 세션 무효화
.cacheTtl long 300000 MFA 캐시 항목의 TTL (밀리초, 기본값: 5분)
.sessionRefreshIntervalMs long 30000 MFA 세션 메타데이터 갱신 간격 (밀리초, 기본값: 30초)
.stateMachineTimeoutMs long 10000 상태 머신 전환 타임아웃 (밀리초, 기본값: 10초)
.deviceRememberDurationMs long 2592000000 신뢰할 수 있는 장치 기억 기간 (밀리초, 기본값: 30일). 기억된 장치에서는 MFA를 건너뜀
YAML 
spring:
  auth:
    mfa:
      sessionTimeoutMs: 600000
      challengeTimeoutMs: 300000
      inactivityTimeout: 900000
      cacheTtl: 300000
      sessionRefreshIntervalMs: 30000
      stateMachineTimeoutMs: 10000
      deviceRememberDurationMs: 2592000000

참고: MFA 생명주기 참조

MFA 재시도 및 잠금

무차별 대입 공격을 방지하기 위한 재시도 제한, 계정 잠금 동작, MFA 시도 간 최소 지연을 제어합니다.

속성 타입 기본값 설명
spring.auth.mfa
.maxRetryAttempts int 5 잠금 전 최대 MFA 검증 재시도 횟수
.accountLockoutDurationMs long 900000 재시도 횟수 초과 후 계정 잠금 기간 (밀리초, 기본값: 15분)
.minimumDelayMs long 500 연속 MFA 검증 시도 간 최소 지연 (밀리초). 타이밍 공격 완화
YAML 
spring:
  auth:
    mfa:
      maxRetryAttempts: 5
      accountLockoutDurationMs: 900000
      minimumDelayMs: 500

참고: MFA 보안 참조

OTP 구성

SMS 및 이메일 전송 채널의 토큰 유효성, 길이, 재전송 간격을 제어하는 일회용 비밀번호(OTP) 설정입니다.

속성 타입 기본값 설명
spring.auth.mfa
.otpTokenValiditySeconds int 300 OTP 토큰 유효 기간 (초, 기본값: 5분)
.otpTokenLength int 6 생성되는 OTP 토큰의 자릿수
.smsResendIntervalSeconds int 60 SMS OTP 재전송 전 최소 간격 (초)
.emailResendIntervalSeconds int 120 이메일 OTP 재전송 전 최소 간격 (초)
YAML 
spring:
  auth:
    mfa:
      otpTokenValiditySeconds: 300
      otpTokenLength: 6
      smsResendIntervalSeconds: 60
      emailResendIntervalSeconds: 120

참고: OTP 구성 참조

상태 머신 풀 및 서킷 브레이커

spring.auth.mfa 네임스페이스 내에서 MFA 상태 머신 객체 풀 크기와 서킷 브레이커 임계값을 제어합니다.

속성 타입 기본값 설명
spring.auth.mfa
.stateMachinePoolSize int 100 객체 풀 내 상태 머신 인스턴스의 최대 수
.stateMachineCacheTtlMs long 300000 캐시된 상태 머신 인스턴스의 TTL (밀리초, 기본값: 5분)
.circuitBreakerFailureThreshold int 5 MFA 서킷 브레이커가 열리기 전 연속 실패 횟수
.circuitBreakerTimeoutSeconds int 30 서킷 브레이커가 반개방 상태로 전환되기 전 열린 상태로 유지되는 시간 (초)
YAML 
spring:
  auth:
    mfa:
      stateMachinePoolSize: 100
      stateMachineCacheTtlMs: 300000
      circuitBreakerFailureThreshold: 5
      circuitBreakerTimeoutSeconds: 30

참고: 상태 머신 참조

MFA 세션 저장소

MFA 흐름을 위한 세션 저장소 구성입니다. HTTP Session, Redis, 인메모리의 세 가지 저장소 백엔드를 지원합니다. 활성 저장소는 autoSelectRepository와 repositoryPriority를 통해 선택됩니다.

속성 타입 기본값 설명
spring.auth.mfa
.sessionStorageType String "http-session" 기본 세션 저장소 백엔드 타입
.autoSelectRepository boolean false 우선순위에 따라 사용 가능한 최적의 저장소를 자동 선택할지 여부
.repositoryPriority String "redis,memory,http-session" 저장소 선택 우선순위를 정의하는 쉼표 구분 목록
.fallbackRepositoryType String "http-session" 기본 저장소를 사용할 수 없을 때의 대체 저장소 타입

HTTP Session

속성 타입 기본값 설명
spring.auth.mfa.httpSession
.enabled boolean true HTTP 세션 기반 MFA 세션 저장소 활성화
.createSessionIfNotExists boolean true HTTP 세션이 존재하지 않을 때 새 세션 생성 여부
.sessionAttributeName String "MFA_SESSION_ID" MFA 세션 식별자를 저장하는 데 사용되는 HTTP 세션 속성 이름

Redis

속성 타입 기본값 설명
spring.auth.mfa.redis
.enabled boolean true Redis 기반 MFA 세션 저장소 활성화
.keyPrefix String "mfa:session:" MFA 세션 항목의 Redis 키 접두사
.cookieName String "MFA_SID" Redis 기반 MFA 세션 식별자를 전송하는 데 사용되는 쿠키 이름
.secureCookie boolean true MFA 세션 쿠키에 HTTPS 전송 필요 여부
.httpOnlyCookie boolean true MFA 세션 쿠키를 클라이언트측 JavaScript에서 접근할 수 없도록 설정할지 여부
.sameSite String "Strict" MFA 세션 쿠키의 SameSite 속성. 값: Strict, Lax, None
.connectionTimeout int 3000 Redis 연결 타임아웃 (밀리초)
.maxRetries int 3 최대 Redis 연결 재시도 횟수

인메모리

속성 타입 기본값 설명
spring.auth.mfa.memory
.enabled boolean true 인메모리 MFA 세션 저장소 활성화
.cleanupIntervalMinutes int 5 만료된 세션 정리 주기 간격 (분)
.maxSessions int 10000 메모리에 유지되는 최대 동시 MFA 세션 수
.enableMetrics boolean true 인메모리 세션 저장소에 대한 메트릭 노출 여부
YAML 
spring:
  auth:
    mfa:
      sessionStorageType: http-session
      autoSelectRepository: false
      repositoryPriority: redis,memory,http-session
      fallbackRepositoryType: http-session
      httpSession:
        enabled: true
        createSessionIfNotExists: true
        sessionAttributeName: MFA_SESSION_ID
      redis:
        enabled: true
        keyPrefix: "mfa:session:"
        cookieName: MFA_SID
        secureCookie: true
        httpOnlyCookie: true
        sameSite: Strict
        connectionTimeout: 3000
        maxRetries: 3
      memory:
        enabled: true
        cleanupIntervalMinutes: 5
        maxSessions: 10000
        enableMetrics: true

참고: MFA 세션 저장소 참조

MFA 팩터 설정

SMS 및 이메일 전송 채널을 포함한 개별 MFA 팩터 제공자의 구성입니다.

SMS 팩터

속성 타입 기본값 설명
spring.auth.mfa.smsFactor
.provider String "default" SMS 전송 제공자 식별자 (예: twilio, aws-sns, default)
.templateId String "mfa_sms_template" SMS OTP 전송을 위한 메시지 템플릿 식별자
.maxDailyAttempts int 10 사용자당 일일 최대 SMS OTP 전송 시도 횟수
.enabled boolean true MFA 등록 및 검증에 SMS 팩터 사용 가능 여부

이메일 팩터

속성 타입 기본값 설명
spring.auth.mfa.emailFactor
.fromAddress String "noreply@company.com" MFA OTP 메시지의 발신자 이메일 주소
.templateId String "mfa_email_template" OTP 전송을 위한 이메일 템플릿 식별자
.maxDailyAttempts int 5 사용자당 일일 최대 이메일 OTP 전송 시도 횟수
.enabled boolean true MFA 등록 및 검증에 이메일 팩터 사용 가능 여부
YAML 
spring:
  auth:
    mfa:
      smsFactor:
        provider: twilio
        templateId: mfa_sms_template
        maxDailyAttempts: 10
        enabled: true
      emailFactor:
        fromAddress: noreply@company.com
        templateId: mfa_email_template
        maxDailyAttempts: 5
        enabled: true

참고: MFA 팩터 구성 참조

모니터링 및 진단

MFA 하위 시스템의 메트릭 수집, 감사 로깅, 상세 진단 로깅을 제어합니다.

속성 타입 기본값 설명
spring.auth.mfa
.detailedLoggingEnabled boolean false MFA 작업에 대한 상세 진단 로깅 활성화. 프로덕션에서는 비활성 유지 권장
.metricsEnabled boolean true 메트릭 엔드포인트(예: Micrometer/Prometheus)를 통한 MFA 메트릭 노출 여부
.auditLoggingEnabled boolean true MFA 이벤트(챌린지, 검증, 실패)를 감사 로그에 기록할지 여부
YAML 
spring:
  auth:
    mfa:
      detailedLoggingEnabled: false
      metricsEnabled: true
      auditLoggingEnabled: true

참고: MFA 모니터링 참조

상태 머신 속성

contexa.identity.statemachine 접두사 아래의 속성으로, StateMachineProperties에 바인딩됩니다. MFA 상태 머신 엔진을 제어합니다.

핵심 설정

속성 타입 기본값 설명
contexa.identity.statemachine
.enabled boolean true 상태 머신 엔진 전체를 활성화 또는 비활성화
.mfa.transitionTimeoutSeconds Integer 10 MFA 흐름 내 개별 상태 전환 타임아웃 (초)
YAML 
contexa:
  identity:
    statemachine:
      enabled: true
      mfa:
        transitionTimeoutSeconds: 10
