AI 엔진 구성

계층형 LLM 전략, 보안 작업 매핑, RAG (Retrieval-Augmented Generation), Advisor 체인, 스트리밍 파이프라인을 포함한 Contexa AI 엔진의 구성 속성입니다.

계층형 LLM 속성

spring.ai.security 접두사 아래의 속성으로, TieredLLMProperties에 바인딩됩니다. 기본 및 백업 모델명, 레이어별 타임아웃, Temperature 설정을 포함한 2계층 분석 파이프라인의 LLM 모델 할당을 구성합니다.

관련 문서: LLM Orchestrator 참조, AI 전략 참조

Layer 1 (컨텍스트 분석)

속성타입기본값설명
spring.ai.security.layer1
.modelStringqwen2.5:14bLayer 1(빠른 분석)의 기본 모델
.backup.modelString—Layer 1 기본 모델을 사용할 수 없을 때의 대체 모델
spring.ai.security.tiered.layer1
.timeout-msInteger30000Layer 1 추론 타임아웃 (ms 단위, 최대 120000)

Layer 2 (전문가 분석)

속성타입기본값설명
spring.ai.security.layer2
.modelStringexaone3.5:latestLayer 2(심층 분석)의 기본 모델
.backup.modelString—Layer 2 기본 모델을 사용할 수 없을 때의 대체 모델
spring.ai.security.tiered.layer2
.timeout-msInteger60000Layer 2 추론 타임아웃 (ms 단위, 최대 120000)

Temperature 기본값

Temperature는 속성으로 직접 구성할 수 없으며 계층별로 프로그래밍 방식으로 설정됩니다: Layer 1 = 0.3 (더 결정론적), Layer 2 = 0.7 (더 창의적).

구성 예제

YAML 
spring:
  ai:
    security:
      layer1:
        model: qwen2.5:14b
        backup:
          model: llama3.2:latest
      layer2:
        model: exaone3.5:latest
        backup:
          model: deepseek-r1:14b
      tiered:
        layer1:
          timeout-ms: 30000
        layer2:
          timeout-ms: 60000

계층형 전략 속성

spring.ai.security.tiered 접두사 아래의 속성으로, TieredStrategyProperties에 바인딩됩니다. RAG 임계값, 캐싱, 프롬프트 구성, 잘림 제한, 각 계층의 벡터 캐시 동작을 포함한 다중 레이어 AI 분석 파이프라인을 제어합니다.

Layer 1 설정

속성 타입 기본값 설명
spring.ai.security.tiered.layer1
.rag.similarityThreshold double 0.5 Layer 1에서 RAG 검색의 최소 유사도
.session.maxRecentActions int 100 분석 컨텍스트의 최대 최근 세션 액션 수
.cache.maxSize int 1000 Layer 1 결과 캐시의 최대 항목 수
.cache.ttlMinutes int 30 Layer 1 캐시 항목의 TTL (분 단위)
.timeout.totalMs long 15000 Layer 1 분석 파이프라인의 전체 타임아웃 (ms)
.timeout.llmMs long 10000 Layer 1 내 LLM 호출 타임아웃 (ms)
.vectorSearchLimit int 5 Layer 1의 최대 벡터 검색 결과 수

Layer 1 프롬프트 설정

속성 타입 기본값 설명
spring.ai.security.tiered.layer1.prompt
.maxSimilarEvents int 3 프롬프트 내 최대 유사 과거 이벤트 수
.maxRagDocuments int 5 프롬프트에 포함되는 최대 RAG 문서 수
.includeEventId boolean false 프롬프트에 원시 이벤트 ID 포함 (ID 노출 가능)
.includeRawTimestamp boolean false 프롬프트에 원시 타임스탬프 포함
.includeRawSessionId boolean false 프롬프트에 원시 세션 ID 포함
.includeFullUserAgent boolean false 전체 User-Agent 포함 (false일 때 잘림 처리)

Layer 2 설정

속성 타입 기본값 설명
spring.ai.security.tiered.layer2
.rag.similarityThreshold double 0.5 Layer 2에서 RAG 검색의 최소 유사도
.timeoutMs long 10000 Layer 2 분석의 전체 타임아웃 (ms)
.enableSoar boolean false Layer 2에서 SOAR 통합 활성화
.ragTopK int 10 Layer 2 분석을 위한 Top-K RAG 결과 수

잘림 설정

프롬프트 크기와 토큰 사용량을 관리하기 위해 다양한 필드의 최대 문자 길이를 제어합니다.

속성 타입 기본값 설명
spring.ai.security.tiered.truncation.layer1
.userAgent int 150 Layer 1 프롬프트에서 User-Agent의 최대 문자 수
.payload int 200 Layer 1 프롬프트에서 페이로드의 최대 문자 수
.ragDocument int 300 Layer 1 프롬프트에서 RAG 문서당 최대 문자 수

벡터 캐시 설정

속성 타입 기본값 설명
spring.ai.security.tiered.vectorCache
.maxSize int 10000 벡터 임베딩 캐시의 최대 항목 수
.expireMinutes int 5 캐시된 벡터 임베딩의 만료 시간 (분)
.enabled boolean true 벡터 임베딩 캐시 활성화 또는 비활성화
.recordStats boolean true 모니터링을 위한 히트/미스 통계 기록

프록시 보안 설정

속성 타입 기본값 설명
spring.ai.security.tiered.security
.trustedProxies List<String> [] X-Forwarded-For용 신뢰할 수 있는 프록시 IP/CIDR
.trustedProxyValidationEnabled boolean true 신뢰 목록에 대한 프록시 헤더 검증
YAML 
spring:
  ai:
    security:
      tiered:
        layer1:
          rag:
            similarityThreshold: 0.5
          session:
            maxRecentActions: 100
          cache:
            maxSize: 1000
            ttlMinutes: 30
          timeout:
            totalMs: 15000
            llmMs: 10000
          prompt:
            maxSimilarEvents: 3
            maxRagDocuments: 5
            includeEventId: false
            includeRawTimestamp: false
            includeRawSessionId: false
            includeFullUserAgent: false
          vectorSearchLimit: 5
        layer2:
          rag:
            similarityThreshold: 0.5
          timeoutMs: 10000
          enableSoar: false
          ragTopK: 10
        truncation:
          layer1:
            userAgent: 150
            payload: 200
            ragDocument: 300
        vectorCache:
          maxSize: 10000
          expireMinutes: 5
          enabled: true
          recordStats: true
        security:
          trustedProxies: []
          trustedProxyValidationEnabled: true

Advisor 속성

contexa.advisor 접두사 아래의 속성으로, ContexaAdvisorProperties에 바인딩됩니다. 보안 Advisor 순서 및 인증 요구 사항을 구성합니다.

속성 타입 기본값 설명
contexa.advisor.security
.enabled boolean true 체인에서 보안 Advisor 활성화
.order int 50 보안 Advisor의 실행 순서 (낮은 값이 먼저 실행)
.requireAuthentication boolean false AI 분석 진행 전 인증된 주체 필요 여부
YAML 
contexa:
  advisor:
    security:
      enabled: true
      order: 50
      requireAuthentication: false

참고: Advisor 체인 참조

RAG 속성

contexa.rag 접두사 아래의 속성으로, ContexaRagProperties에 바인딩됩니다. 기본 검색 매개변수와 AI Lab 배치 처리를 포함한 Retrieval-Augmented Generation 파이프라인을 구성합니다.

기본 검색 매개변수

속성 타입 기본값 설명
contexa.rag.defaults
.similarityThreshold double 0.7 RAG 쿼리의 기본 최소 코사인 유사도 임계값
.topK int 10 RAG 쿼리당 반환되는 기본 Top-K 문서 수

AI Lab 설정

속성 타입 기본값 설명
contexa.rag.lab
.batchSize int 50 AI Lab 문서 처리 작업의 배치 크기
.validationEnabled boolean true 벡터 스토어 수집 전 문서 검증 활성화
.enrichmentEnabled boolean true 문서 처리 중 메타데이터 보강 활성화
.topK int 100 AI Lab 유사도 검색의 Top-K 제한
.similarityThreshold double 0.75 AI Lab 문서 매칭의 유사도 임계값
YAML 
contexa:
  rag:
    defaults:
      similarityThreshold: 0.7
      topK: 10
    lab:
      batchSize: 50
      validationEnabled: true
      enrichmentEnabled: true
      topK: 100
      similarityThreshold: 0.75

참고: RAG 파이프라인 참조

스트리밍 속성

contexa.streaming 접두사 아래의 속성으로, StreamingProperties에 바인딩됩니다. 프로토콜 마커와 버퍼 설정을 포함한 실시간 LLM 응답 전달에 사용되는 SSE (Server-Sent Events) 스트리밍 파이프라인을 구성합니다.

속성 타입 기본값 설명
contexa.streaming
.finalResponseMarker String StreamingProtocol 상수 SSE 스트림에서 마지막 응답 청크를 나타내는 마커 문자열
.markerBufferSize int 100 스트림 파서의 마커 감지를 위한 내부 버퍼 크기
YAML 
contexa:
  streaming:
    markerBufferSize: 100

참고: 스트리밍 파이프라인 참조

PgVector 스토어 속성

spring.ai.vectorstore.pgvector 접두사 아래의 속성으로, Spring AI의 PGVector 자동 구성에서 제공됩니다. 계층형 전략과 RAG 파이프라인 모두에서 사용되는 PostgreSQL 기반 벡터 스토어를 구성합니다.

주요 설정에는 테이블명, 임베딩 차원, 인덱스 타입(HNSW 또는 IVFFlat), 거리 함수, 스키마 초기화 동작이 포함됩니다. 전체 속성 참조는 전용 페이지를 확인하세요.

YAML 
spring:
  ai:
    vectorstore:
      pgvector:
        index-type: HNSW
        distance-type: COSINE_DISTANCE
        dimensions: 384
        initialize-schema: true

참고: PgVector 구성 참조
