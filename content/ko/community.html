커뮤니티

Contexa 커뮤니티에 참여하세요. 코드 기여, 이슈 보고, 질문, 그리고 Spring을 위한 AI 네이티브 보안의 미래를 함께 만들어 가세요.

기여하기

Contexa는 오픈 소스 프로젝트이며, 버그 수정과 문서 개선부터 새로운 기능과 아키텍처 개선까지 모든 종류의 기여를 환영합니다.

1

Fork & Clone

GitHub에서 리포지토리를 Fork하고 로컬 머신에 Clone하세요.

Shell 
git clone https://github.com/your-username/contexa.git
cd contexa
git checkout -b feature/your-feature-name
2

빌드 & 테스트

변경하기 전에 기존의 모든 테스트가 통과하는지 확인하세요. Contexa는 Gradle을 사용하여 빌드합니다.

Shell 
./gradlew build
./gradlew test
3

Pull Request 제출

브랜치를 Push하고 main 브랜치에 대한 Pull Request를 열어주세요. 변경 사항을 명확하게 설명하고 관련 이슈를 참조하세요.

GitHub에서 연결하기

Contexa 프로젝트는 GitHub에서 운영됩니다. 참여할 수 있는 주요 채널은 다음과 같습니다.

이슈

버그 보고, 기능 요청, 또는 기존 이슈를 확인하세요. 라벨을 사용하여 모듈이나 우선순위별로 필터링할 수 있습니다.

커뮤니티 vs 엔터프라이즈

Contexa Community Edition은 Apache License 2.0에 따라 무료 오픈 소스로 제공됩니다. Enterprise Edition은 프로덕션 환경의 핵심 워크로드를 위한 추가 기능을 제공합니다.

기능 Community Enterprise
Identity DSL (Form, REST, OTT, Passkey, MFA) 포함 포함
XACML Policy Engine & @Protectable 포함 포함
Zero Trust Actions (5가지 액션) 포함 포함
HCAD AI Engine (Hot Path + Cold Path) 포함 포함
Shadow Mode (2단계 마이그레이션) 포함 포함
SSE Streaming & PgVector RAG 포함 포함
Standalone & Distributed 모드 포함 포함
SOAR (Security Orchestration, Automation & Response) — 포함
MCP Server 통합 — 포함
Admin Console & Policy Studio 제한적 전체
우선 지원 & SLA 커뮤니티 전담
라이선스 Apache 2.0 Commercial

Enterprise Edition에 관한 문의는 아래 연락 채널을 통해 연락해 주세요.

문의

Contexa에 대한 질문이 있거나, 통합에 도움이 필요하거나, Enterprise Edition에 관심이 있으신가요? 아래 채널을 통해 연락해 주세요.
